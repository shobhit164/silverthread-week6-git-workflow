# Git Workflow Summary

The team used a lightweight branching workflow to support parallel development:

1. Start from a stable `main` branch.
2. Create a focused branch for each task.
3. Commit small, descriptive changes.
4. Open a pull request for review.
5. Merge reviewed work into `main`.
6. Resolve conflicts when two branches edit the same area.

This workflow helped the team work independently without overwriting each other. It also made each change easier to review because every branch had a clear purpose.
